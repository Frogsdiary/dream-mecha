Documentation

Button Components

Primary Button
Standard action button with white background and black text. Used for primary actions like "Save", "Purchase", "Expand Grid".
.btn { background: var(--accent); color: var(--bg-primary); border: none; padding: var(--dot-4) var(--dot-6); cursor: pointer; font-weight: bold; text-transform: uppercase; letter-spacing: var(--dot); transition: all 0.2s ease; font-size: var(--font-size-sm); font-family: inherit; }
Secondary Button
Alternative button style with transparent background and white border. Used for secondary actions or cancel operations.
.btn.secondary { background: transparent; color: var(--text-primary); border: 1px solid var(--text-secondary); }

Grid System

Grid Container
CSS Grid-based system with 40px cells and 4px gaps. Supports drag-and-drop operations and visual feedback states.
.grid { display: grid; gap: var(--grid-gap); background: rgba(34, 34, 34, 0.8); padding: var(--dot-4); border: 1px solid var(--text-secondary); } .grid-cell { width: var(--grid-cell); /* 40px */ height: var(--grid-cell); background: rgba(17, 17, 17, 0.9); border: 1px solid var(--text-secondary); cursor: pointer; transition: all 0.2s ease; }
Grid States
Different visual states for grid cells: empty, occupied, selected, hover, drag-over, and ghost preview.
.grid-cell.occupied { border-color: var(--accent); } .grid-cell.selected { border: 3px solid var(--accent); box-shadow: 0 0 10px var(--accent); } .grid-cell.ghost { background: rgba(255, 255, 255, 0.3) !important; border: 2px dashed var(--accent) !important; opacity: 0.8; }

Stat Color System

Color Coding
Consistent color scheme for different stat types: HP (red), Attack (orange), Defense (blue), Speed (green).
.stat-hp { background-color: #ff4444; } .stat-attack { background-color: #ff8800; } .stat-defense { background-color: #4444ff; } .stat-speed { background-color: #44ff44; }

API & Database

Web API Endpoints

GET /api/player/{player_id}/grid
Get player's current grid state
POST /api/player/{player_id}/grid
Update player's grid with new piece placement
GET /api/shop
Get current daily shop inventory
POST /api/shop/purchase
Purchase item from shop
GET /api/game/status
Get current game state and voidstate
GET /api/daily/{date}
Get daily content for specific date
GET /health
Health check endpoint

Player Data Structure
Core player data stored in JSON format per player file.

Error Handling

HTTP Status Codes
Standard HTTP response codes for API endpoints.
200 - Success 201 - Created (new player/mecha) 400 - Bad Request (invalid data) 401 - Unauthorized (no auth) 404 - Not Found (player/piece not found) 409 - Conflict (already exists) 422 - Unprocessable Entity (validation error) 500 - Internal Server Error
Error Response Format
Consistent error response structure.
{ "error": "Error message", "code": "ERROR_CODE", "details": "Additional details", "timestamp": "ISO datetime" }
Common Error Codes
Standard error codes used throughout the system.
PLAYER_NOT_FOUND - Player doesn't exist INSUFFICIENT_ZOLTANS - Not enough currency INVALID_GRID_PLACEMENT - Piece can't be placed MECHA_NOT_LAUNCHABLE - HP too low to launch SHOP_ITEM_UNAVAILABLE - Item already sold DAILY_LIMIT_EXCEEDED - Purchase limit reached
